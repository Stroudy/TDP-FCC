HTML

What is HTML

HTML, the backbone of web documents, shapes their content and structure. It works alongside CSS and JavaScript, facilitating webpage design and functionality. Browsers interpret HTML, transforming it into multimedia pages, initially designed to convey both structure and appearance cues.

HTML elements serve as the foundation of webpage construction, allowing the embedding of various content types. Tags denote the structural semantics of text elements, influencing the interpretation of page content. While some tags directly introduce content, others provide contextual information, guiding browsers in rendering the page.

JavaScript integration within HTML plays a pivotal role in shaping webpage behavior and content. Meanwhile, CSS dictates the visual presentation, defining the look and layout of the content. The World Wide Web Consortium (W3C), overseeing HTML and CSS standards, advocates for CSS adoption over explicit presentational HTML, emphasizing separation of structure and presentation since 1997.

HTML5, an evolved version of HTML, revolutionizes multimedia integration. It leverages elements like canvas and JavaScript to display video and audio content. This advancement signifies a paradigm shift in web development, enhancing interactivity and media integration while maintaining standards compliance with W3C recommendations.

History

HTML's origins trace back to physicist Tim Berners-Lee's 1980s proposal for ENQUIRE at CERN, evolving into a memo on an Internet-based hypertext system in 1989. Berners-Lee specified HTML and developed browser and server software in 1990, with early funding efforts alongside Robert Cailliau.

The first public description of HTML emerged in late 1991, detailing 18 elements influenced by SGMLguid. HTML, a markup language, orchestrates web content composition and interpretation by browsers. CSS enhances default browser-rendered characteristics, evolving from SGML-based text formatting commands used since the 1960s.

Berners-Lee viewed HTML as an SGML application, later formalized by the IETF in 1993. Drafts such as "HTML+" and "HTML 2.0" emerged, reflecting evolving web standards. The HTML Working Group, formed in 1994, eventually produced HTML 2.0, setting a standard for future implementations.

Stalled development under the IETF led to the W3C's stewardship in 1996, ensuring HTML's evolution. HTML 4.01 became an international standard in 2000. Subsequent efforts culminated in HTML5's development, addressing contemporary web needs and finalized as a joint deliverable by the WHATWG and W3C in 2014.

Attributes

  • The id attribute assigns unique identifiers to HTML elements, enabling targeted style and script manipulation. When added to a page's URL, it provides global identification, facilitating direct referencing.
  • The class attribute categorizes similar elements for semantic or presentational purposes. It allows grouping and styling elements accordingly. Microformats utilize class attributes for semantic purposes. Multiple class values can be specified for an element.
  • The style attribute assigns presentational properties to an element. While using id or class attributes is preferred for stylesheet selection, the style attribute offers simplicity for specific or ad hoc styling needs.
  • The title attribute is used to attach a subtextual explanation to an element. In most browsers this attribute is displayed as a tooltip.

    CSS styles HTML/XML documents, separating content and presentation for improved accessibility and flexibility. It allows sharing formatting across pages, caching for faster load times, and different styling for various rendering methods. W3C maintains CSS standards, applicable to multiple markup languages.

    A declaration block encloses declarations within braces, each consisting of a property, colon, and value. Properties, defined in the CSS standard, affect elements differently. Values can be keywords, numerical measurements, or color specifications, utilizing various units for precise styling.

    CSS revolutionized web design by moving presentational attributes from HTML markup to separate style sheets. This simplifies HTML, promotes consistency across devices, and enables efficient reuse of styles. CSS separates presentation from structure, enhancing maintainability and accessibility while deprecating presentational HTML markup.

    CSS allows flexible styling of web content, sourced from browser defaults, user preferences, or designers. Styles can be inline, within HTML, or in external .css files, enhancing reusability, maintainability, and site performance. This separation ensures accessibility and readability, accommodating diverse user needs and devices.

    CSS inheritance propagates property values from ancestors to descendants based on the document tree. Text-related properties are typically inherited, while box-related ones are not. This feature reduces redundancy in style sheets, promoting shorter CSS code. Unlike class-based inheritance in programming languages, CSS inheritance doesn't allow defining modified classes for reuse.

    Specificity in CSS measures the weight of rules, determining which styles apply when multiple rules conflict. Simple selectors have a specificity of 1, while class selectors are 1,0, and ID selectors are 1,0,0. Specificity values don't carry over; commas separate the "digits."

    CSS evolved to address the need for web style sheets, distinct from SGML. Unlike existing languages like DSSSL and FOSI, CSS allowed for cascading styles from multiple sources, meeting the demand for diverse web styling.

    HTML's evolution accommodated diverse stylistic capabilities to meet developer demands, granting more design control but resulting in complex HTML. Browser variations like ViolaWWW and WorldWideWeb caused inconsistency in site appearance, limiting user control. Tim Berners-Lee's browser/editor had hardcoded style sheets, impeding web document linking. Robert Cailliau sought to separate structure from presentation for varied styling purposes

    Improving web presentation sparked interest, leading to nine style sheet proposals on the www-style mailing list. Notably, "Cascading HTML Style Sheets" and "Stream-based Style Sheet Proposal" influenced CSS's development. Lie collaborated with Yves Lafon to implement CSS in Arena, while Bert Bos implemented SSP in Argo. Lie and Bos collaborated on CSS, removing the 'H' to denote applicability to various markup languages.
